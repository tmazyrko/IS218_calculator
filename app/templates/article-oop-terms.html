OOP Terms

Here you can find some definitions for commonly used terms in the realm of object-oriented programming.

Instantiation
The realization of a predefined object; an instantiated object is given a name and created in memory or on disk using the structure described within a class declaration.

Object
A value in memory referenced by an identifier. Can be a variable, a data structure, a function, a method, etc.

Class
A program-code-template of a specified object; provides initial values for the object’s state and implementations of its behavior.

Function
A named section of a program that performs a specific task and can return a value.

Method
A type of function that is part of a class and has access to the class’ member variables.

Namespace
A grouping of symbols (such as variables and functions) that help avoid naming conflicts; a context where a set of names is relevant.

Constructor
A special type of method that is called to create an object and prepare it for use, often accepting arguments that the constructor uses to set required member variables.

Fixture
A method that sets up a system for the software testing process by initializing it and satisfying any preconditions the testing may depend on.

Type Hinting
A formal way of statically indicating the type of a value in Python code. Specified in PEP 484 and introduced in Python 3.5.

Python Example

def greet(name: str) -> str:
    return "Hello, " + name

Type Casting
A way of specifying a type onto another variable.

Python Example

x = int(1)   # x will be 1
y = int(2.8) # y will be 2
z = int("3") # z will be 3

x = float(1)     # x will be 1.0
y = float(2.8)   # y will be 2.8
z = float("3")   # z will be 3.0
w = float("4.2") # w will be 4.2

x = str("s1") # x will be 's1'
y = str(2)    # y will be '2'
z = str(3.0)  # z will be '3.0'

Unit Testing
A software testing method in which individual units of code are tested against known conditions and results to determine whether the code is fit for use.

Static
A keyword indicating that the variable or method marked as such is available at the class level, meaning an instance of the class does not need to be created to access it. In Python, a static method is denoted by the decorator @staticmethod.

Class Method
A method that cannot modify the state of any one object instance, but can modify class state that applies across all instances of that class. In Python, a class method is denoted by the decorator @classmethod.

Property
A special type of class member that uses a field-like syntax for reading and writing, but property reads and writes are translated to ‘getter’ and ‘setter’ method calls. In Python, the @property decorator can be used to define a property.

Design pattern
A typical solution to commonly occurring issues in software design; can be likened to a pre-made blueprint that can be adapted to solve a recurring design problem in your code.

Factory
A method that provides an interface for creating objects in a superclass, which also allows subclasses to alter the type of object that will be created.

Facade
A structural design pattern that provides a simplified interface to a library, framework, or other complex set of classes.
